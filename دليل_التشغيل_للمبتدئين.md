# 🎯 دليل التشغيل الكامل للمبتدئين

## ✅ خطوة 1: تنظيف شامل

### من Docker Desktop (الطريقة الأسهل):

1. **افتح Docker Desktop**
2. انتقل إلى **"Images"** (الصور)
3. **احذف كل صورة** تحتوي على:
   - `ecommerce-store-backend`
   - `ecommerce-store-frontend`
   
   👉 اضغط 🗑️ بجانب كل صورة

4. انتقل إلى **"Containers"**
5. **احذف كل حاوية** (Container) موجودة

---

## 🚀 خطوة 2: البناء والتشغيل

### الطريقة 1: من PowerShell (موجه الأوامر) - الأفضل ✅

#### أ. افتح PowerShell:
```
Win + X → ثم اختر "Windows PowerShell"
```

#### ب. انتقل لمجلد المشروع:
```powershell
cd C:\Users\hp\Desktop\projet2\ecommerce-store
```

#### ج. نظّف كل شيء:
```powershell
docker-compose down -v
docker system prune -f
```

#### د. ابنِ وشغّل (سيأخذ 5-10 دقائق أول مرة):
```powershell
docker-compose up --build
```

**⏱️ انتظر حتى ترى:**
```
✅ backend  | INFO: Application startup complete
✅ frontend | ➜ Local: http://localhost:5173/
```

#### هـ. افتح المتصفح:
```
http://localhost:5173
```

---

### الطريقة 2: من Docker Desktop GUI (أقل كفاءة لكن أسهل للمبتدئين)

1. في Docker Desktop، اضغط **"Containers"** من القائمة
2. في شريط البحث، اكتب مسار المشروع أو اضغط **"+"** لإضافة مشروع جديد
3. **لكن** هذه الطريقة لن تعمل جيداً لأول مرة - استخدم PowerShell!

## 🔍 كيف تتأكد أنك تعمل على النسخة المصححة؟

### تحقق من هذه الملفات:

#### 1. افتح: `backend/app/models/product.py`
**يجب أن يحتوي السطر 2 على:**
```python
from sqlalchemy.orm import relationship
```

#### 2. افتح: `backend/app/api/v1/schemas.py`
**يجب أن يكون السطر 108 قبل السطر 119:**
```python
class SupplierProductResponse(BaseModel):  # السطر 108
    ...

class ProductResponse(BaseModel):  # السطر 119
```

#### 3. افتح: `docker-compose.yml`
**يجب ألا يحتوي على `healthcheck`**

## ⚡ إذا واجهتك مشاكل:

### المشكلة: PowerShell لا يقبل الأوامر
**الحل:**
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### المشكلة: Docker لا يعمل
**الحل:**
1. تأكد أن Docker Desktop يعمل (أيقونة الحوت في شريط المهام)
2. إذا لم يعمل، أعد تشغيل Docker Desktop

### المشكلة: "port already in use"
**الحل:**
```powershell
docker-compose down
# ثم
docker-compose up --build
```
