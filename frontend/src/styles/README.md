# مبادئ تنسيق الواجهات (Style Guide)

## 1) ملف المتغيرات (Tokens)

- استخدم `tokens.css` لتخزين جميع المتغيرات (ألوان، أحجام، تباعد، زوايا) بحيث تصبح مركزية وسهلة التعديل.

## 2) ملفان عامان

- `styles.css`: الأنماط الأساسية والعالمية، عناصر الواجهة العامة (مثل `btn`، `header-nav`, `product-card`).
- `styles/pages.css`: أنماط تخطيطية مخصصة للصفحات تُستخدم فقط عندما تحتاج الصفحة إلى تخطيطات خاصة.

## 3) أنماط المكوّنات (Components)

- لكل مكوّن قابل لإعادة الاستخدام استخدم CSS Modules أو ملف CSS خاص به للحفاظ على نطاق الأنماط ومنع التسرب.
- يفضّل تصنيف أنماط المكوّنات في `styles/components` أو داخل نفس مجلد المكوّن.

## 4) متى نستخدم ملف صفحة منفصل؟

- Use case: الصفحة تحتاج تخطيطاً خاصاً، أو أنماط محددة للغاية لا تُستخدم في صفحات أخرى.
- لا تنسخ أنماط المكوّنات داخل صفحات — استبدل إعادة الاستخدام بمكوّنات إذا أمكن.

## 5) نصائح إضافية

- استعمل المتغيرات (tokens) بدلاً من القيم الثابتة.
- تجنّب التكرار: اجعل القواعد العامة في `styles.css` أو كواسمات للمكوّنات.
- اجعل `pages.css` خفيفاً: فقط تخطيطات وفضاءات مختلفة/قواعد خاصة بالصفحة.
- استخدم CSS Modules للمكوّنات المعقدة التي تحتاج حجاب نطاق (scoping).

**باختصار:** تعتمد أفضل ممارسة على مزيج من (Tokens, Global Styles, Component Scoped Styles, Page-level Overrides).

## 6) دعم الثيم (Theme)

- التطبيق يدعم الوضع الداكن عبر متغيرات في `tokens.css` (تحت `.dark-mode`).
- استخدم `ThemeToggle` في الهيدر للتبديل بين `light` و `dark`؛ الحالة تحفظ في `localStorage` تلقائياً.
- لتطبيق تغيّر سلس عند التبديل، أضفنا انتقالات (`transition`) على خصائص الخلفية والنص في `styles.css`.
